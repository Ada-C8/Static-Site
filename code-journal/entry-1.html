Automated Testing

August 18, 2017

It's almost the end of the 2nd week at Ada and things are far from slowing down!

It's funny how in the X amount of months I spent learning programming, we've already covered those topics in just the last week.

Today we learned how to do automated testing using Minitest and Test-Driven Development.

Test-Driven Development is done by writing bare minimum tests for each functionality you want your code to have FIRST and after seeing the error, only then writing the implementation of your code. That's why it's called "Test-Driven" Development.

You alternate between writing tests and writing code. But it's important to look at the errors, no matter how jarring they may be.
