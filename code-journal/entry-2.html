No More Excruciatingly Explicit Variable Names: Modules and Namespacing

August 22, 2017

I try to be very explicit when I name my variables. So explicit that the character count gods must be shaking their head in disappointment whenever I name a variable something like 'id_customer_bank_account_number' with another variable 'id_employee_bank_account_number'.

So imagine finding out that there was a way so that you can name variables with the same name but under different contexts?

Enter Modules and using them as a way to bundle related code together and avoid naming collision. This bundling is called Namespacing.
